TabularLab 中文操作说明书

TabularLab 是一个面向表格数据的本地机器学习分析工具，可作为桌面 App 使用，也可以在浏览器中打开。它帮助用户完成数据导入、字段选择、预处理、模型训练、评估、可视化、预测和结果导出。

软件名称 TabularLab : A lightweight toolkit for tabular machine learning.
作者 Dr. Bin Cao / 曹斌
版本 v1.0.0，发布日期 2026-06-29
下载 App Windows / macOS 下载页面
反馈 如果有改进建议，请在 GitHub Issues 提交。

1. 界面目的

这个界面的目标是让用户无需安装 Python、R 或后端服务，也可以快速完成中小型表格数据的机器学习分析。使用桌面 App 时，依赖和示例数据已随 App 打包；使用浏览器方式时，计算在本地浏览器中完成。它适合教学演示、材料数据探索、原型分析、模型对比和快速实验。

  • 降低表格机器学习实验门槛：上传文件、选择任务、点击运行即可获得结果。
  • 统一处理回归、分类和聚类三类常见任务。
  • 把数据预处理、训练测试划分、交叉验证、指标、图表和导出放在同一个界面中。
  • 帮助用户理解每一步设置对最终模型输出的影响。

2. 快速开始

打开界面

推荐从 发布页面 下载 Windows 或 macOS App 后直接运行。也可以在浏览器中打开项目根目录下的 index.html；如果使用浏览器方式，建议通过本地静态服务器访问，例如 python3 -m http.server 8000 后打开 http://localhost:8000/。

导入数据

可以拖入 CSV、TSV、TXT、XLSX 或 XLS 文件，也可以从“默认数据”下拉框选择示例数据并点击“加载示例数据”。

选择任务

在“任务类型”中选择回归、分类或聚类。回归用于预测连续数值，分类用于预测类别标签，聚类用于无监督发现数据分组。

设置字段角色

在“字段角色”表中把每一列设置为特征、目标或忽略。聚类任务没有目标列，通常把可用于分群的列设为特征。

配置处理与训练

选择缺失值策略、文本编码方式、模型和模型超参数。非聚类任务还可以设置测试集比例、随机种子、交叉验证和折数。

运行并查看结果

点击“运行分析”，然后在“结果”“可视化”“预测”“数据表”标签页查看指标、图表、预测和导出文件。

3. 数据源与数据要求

支持格式

  • .csv、.tsv、.txt：文本分隔表格。
  • .xlsx、.xls：Excel 表格，使用项目内置 SheetJS 解析。

数据建议

  • 第一行应为字段名，字段名最好唯一且简洁。
  • 回归目标列应为数值，例如强度、价格、容量、温度等。
  • 分类目标列应为类别标签，例如材料类别、合格/不合格、工艺窗口标签等。
  • 聚类任务不需要目标列，特征列应能表达样本差异。
  • 缺失值可以保留，界面会根据选择的策略处理。

4. 主要功能

数据导入与预览

导入后显示行数、列数、缺失值、数值列、类别列和字段摘要，并可以预览前 N 行数据。

字段角色管理

每列可以设置为特征、目标或忽略。目标列支持多选，适合多目标回归或多目标分类。

缺失值处理

支持数值均值/类别众数、数值中位数/类别众数、0/Unknown 三种策略。

类别编码

支持独热编码和序号编码，用于把文本或符号列转换为模型可使用的数值特征。

模型超参数

根据所选模型显示关键参数，例如 K、树深、树数量、学习率、正则化强度、训练轮数、DBSCAN 的 eps 和 minPts。

评估与导出

支持测试集评估、交叉验证 OOF 评估、图表保存、模型结果 JSON、预测 CSV 和处理后数据 CSV 导出。

5. 任务与模型说明

回归任务

回归用于预测连续数值。可选模型包括 Linear/Ridge Regression、Lasso、ElasticNet、Linear SVR、Decision Tree、Random Forest、Gradient Boosting 和 KNN Regression。常见指标包括 MAE、RMSE 和 R2。

分类任务

分类用于预测离散类别。可选模型包括 Softmax Logistic、Linear SVM、Decision Tree、Random Forest、Gaussian Naive Bayes 和 KNN Classifier。常见指标包括 Accuracy、Precision、Recall、F1 和混淆矩阵。

聚类任务

聚类用于无监督分组。可选模型包括 KMeans、DBSCAN 和 Agglomerative Clustering。聚类不需要目标列，也不需要训练/测试集划分；界面会使用全部选中特征数据进行聚类。

6. 标签页使用说明

  • 概览：查看数据集规模、缺失值情况、字段类型、唯一值、均值、标准差、最小值和最大值。
  • 可视化：绘制直方图、散点图、相关性热力图和目标关系图，并保存为 PNG。
  • 结果：查看模型指标、当前评估模式、结果图、分类报告、聚类投影或线性系数。
  • 预测：输入单条样本进行预测，或上传新文件做批量预测并下载结果。
  • 数据表：预览原始数据，并导出经过缺失值和编码处理后的数据。
  • 关于：查看软件标题、作者、版本、发布日期、引用格式和 BibTeX。

7. 保存与引用

  • 图表可保存为 PNG 文件。
  • 模型结果可保存为 JSON，包含任务类型、目标列、特征列、评估模式和指标。
  • 预测结果可保存为 CSV，便于后续分析或报告。
  • 处理后数据可保存为 CSV，用于复现实验或导入其他软件。
  • 引用信息可在“版本与引用”或“关于”标签页复制。

8. 注意事项

  • TabularLab 面向中小型数据和快速实验，不建议直接用于百万行级生产训练。
  • 所有计算在浏览器中执行，浏览器性能会影响运行速度。
  • 测试集指标只代表划分出的测试集表现；交叉验证指标代表 OOF 预测表现。
  • 批量预测文件必须包含训练时使用的全部特征列，其他列会保留在输出文件中。

TabularLab English User Manual

TabularLab is a local machine learning interface for tabular data. It can be used as a desktop app or opened in a browser, and helps users import data, define columns, preprocess features, train models, evaluate results, visualize outputs, make predictions, and export files.

Software Title TabularLab : A lightweight toolkit for tabular machine learning.
Author Dr. Bin Cao / 曹斌
Version v1.0.0, released on 2026-06-29

1. Purpose

The purpose of this interface is to make tabular machine learning accessible without installing Python, R, or a backend service. In the desktop app, dependencies and example datasets are bundled; in browser use, calculations run locally in the browser. The tool is suitable for teaching, materials-data exploration, prototyping, model comparison, and quick experiments.

  • Lower the entry barrier for tabular ML workflows: load a file, choose a task, and run analysis.
  • Support regression, classification, and clustering in one interface.
  • Combine preprocessing, train/test splitting, cross validation, metrics, charts, predictions, and exports.
  • Help users understand how each setting affects model outputs.

2. Quick Start

Open the interface

Download the Windows or macOS app from the release page and run it directly. You can also open index.html in a browser; for browser use, a local static server is recommended, such as python3 -m http.server 8000, then open http://localhost:8000/.

Import data

Drop a CSV, TSV, TXT, XLSX, or XLS file into the upload area, or choose a default dataset and click “Load Sample”.

Select a task

Choose regression, classification, or clustering. Regression predicts continuous values, classification predicts labels, and clustering discovers groups without target labels.

Assign column roles

In the column-role table, set each column as a feature, target, or ignored column. Clustering does not require a target column.

Configure processing and training

Choose a missing-value strategy, text encoding method, model, and model hyperparameters. For supervised tasks, configure test size, random seed, cross validation, and folds.

Run and inspect results

Click “Run Analysis”, then inspect metrics, charts, predictions, and exports in the Results, Visuals, Predict, and Data Table tabs.

3. Data Sources and Requirements

Supported formats

  • .csv, .tsv, .txt: delimited text tables.
  • .xlsx, .xls: Excel files parsed by the bundled SheetJS library.

Data recommendations

  • The first row should contain clear and unique column names.
  • Regression targets should be numeric, such as strength, price, capacity, or temperature.
  • Classification targets should be labels, such as material class, pass/fail, or process-window status.
  • Clustering tasks do not need a target column; selected features should describe meaningful sample differences.
  • Missing values can remain in the file. They are handled according to the selected strategy.

4. Main Functions

Data import and preview

After import, the interface shows rows, columns, missing values, numeric columns, categorical columns, column summaries, and the first N preview rows.

Column role management

Each column can be marked as feature, target, or ignored. Multiple targets are supported for multi-target regression or classification.

Missing-value handling

Supported strategies include numeric mean/category mode, numeric median/category mode, and 0/Unknown.

Categorical encoding

One-hot and ordinal encoding convert text or symbolic columns into numeric features for machine learning models.

Model hyperparameters

The interface displays key parameters for the selected model, such as K, tree depth, tree count, learning rate, regularization, epochs, DBSCAN eps, and minPts.

Evaluation and export

TabularLab supports test-set evaluation, cross-validation OOF evaluation, chart export, model-result JSON, prediction CSV, and processed-data CSV.

5. Tasks and Models

Regression

Regression predicts continuous numeric values. Available models include Linear/Ridge Regression, Lasso, ElasticNet, Linear SVR, Decision Tree, Random Forest, Gradient Boosting, and KNN Regression. Common metrics include MAE, RMSE, and R2.

Classification

Classification predicts discrete labels. Available models include Softmax Logistic, Linear SVM, Decision Tree, Random Forest, Gaussian Naive Bayes, and KNN Classifier. Common outputs include Accuracy, Precision, Recall, F1, and a confusion matrix.

Clustering

Clustering discovers groups without labels. Available models include KMeans, DBSCAN, and Agglomerative Clustering. It does not need a target column or train/test split; all selected feature rows are used for clustering.

6. Tab Guide

  • Overview: Inspect dataset size, missing values, column types, unique values, mean, standard deviation, minimum, and maximum.
  • Visuals: Draw histograms, scatter plots, correlation heatmaps, and target plots, then save charts as PNG files.
  • Results: Inspect model metrics, evaluation mode, result charts, classification report, clustering projection, or linear coefficients.
  • Predict: Enter one sample for prediction, or upload a new file for batch prediction and download the output.
  • Data Table: Preview raw data and export processed data after missing-value handling and encoding.
  • About: View software title, author, version, release date, citation, and BibTeX.

7. Saving and Citation

  • Charts can be saved as PNG files.
  • Model results can be saved as JSON with task type, targets, features, evaluation mode, and metrics.
  • Prediction outputs can be saved as CSV for downstream analysis or reporting.
  • Processed data can be saved as CSV for reproducibility or use in other tools.
  • Citation text can be copied from the Version & Citation panel or the About tab.

8. Notes

  • TabularLab is designed for small to medium datasets and quick experiments, not million-row production training.
  • Runtime depends on the local computer. Browser use is affected by browser performance, while desktop app use is affected by the bundled Electron runtime and system resources.
  • Test metrics represent the held-out test split. Cross-validation metrics represent out-of-fold predictions.
  • Batch prediction files must contain every feature column used during training. Other columns are preserved in the output.
