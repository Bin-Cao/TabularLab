TabularLab 中文操作说明书

TabularLab 是一个面向表格数据的本地机器学习分析工具，可作为桌面 App 使用，也可以在浏览器中打开。它帮助用户完成数据导入、字段选择、预处理、模型训练、评估、可视化、预测和结果导出。

软件名称 TabularLab : A lightweight toolkit for tabular machine learning.

下载 App Windows / macOS 下载页面

1. 界面目的

这个界面的目标是让用户无需安装 Python、R 或后端服务，也可以快速完成中小型表格数据的机器学习分析。使用桌面 App 时，依赖和示例数据已随 App 打包；使用浏览器方式时，计算在本地浏览器中完成。它适合教学演示、材料数据探索、原型分析、模型对比和快速实验。

  • 降低表格机器学习实验门槛：上传文件、选择任务、点击运行即可获得结果。
  • 统一处理回归、分类和聚类三类常见任务。
  • 把数据预处理、训练测试划分、交叉验证、指标、图表和导出放在同一个界面中。
  • 帮助用户理解每一步设置对最终模型输出的影响。

2. 快速开始

打开界面

推荐从 发布页面 下载 Windows 或 macOS App 后直接运行。也可以在浏览器中打开项目根目录下的 index.html；如果使用浏览器方式，建议通过本地静态服务器访问，例如 python3 -m http.server 8000 后打开 http://localhost:8000/。

导入数据

可以拖入 CSV、TSV、TXT、XLSX 或 XLS 文件，也可以从“默认数据”下拉框选择示例数据并点击“加载示例数据”。

选择任务

在“任务类型”中选择回归、分类或聚类。回归用于预测连续数值，分类用于预测类别标签，聚类用于无监督发现数据分组。

设置字段角色

在“字段角色”表中把每一列设置为特征、目标或忽略。聚类任务没有目标列，通常把可用于分群的列设为特征。

配置处理与训练

选择缺失值策略、文本编码方式、模型和模型超参数。非聚类任务还可以设置测试集比例、随机种子、交叉验证和折数。

运行并查看结果

点击“运行分析”，然后在“结果”“可视化”“预测”“数据表”标签页查看指标、图表、预测和导出文件。

3. 数据源与数据要求

支持格式

  • .csv、.tsv、.txt：文本分隔表格。
  • .xlsx、.xls：Excel 表格，使用项目内置 SheetJS 解析。

数据建议

  • 第一行应为字段名，字段名最好唯一且简洁。
  • 回归目标列应为数值，例如强度、价格、容量、温度等。
  • 分类目标列应为类别标签，例如材料类别、合格/不合格、工艺窗口标签等。
  • 聚类任务不需要目标列，特征列应能表达样本差异。
  • 缺失值可以保留，界面会根据选择的策略处理。

4. 字段角色：特征、目标和忽略

机器学习表格通常由“行”和“列”组成。每一行是一条样本，每一列是一种信息。运行模型前，需要告诉 TabularLab 每一列在任务中扮演什么角色。

特征 Feature

特征是模型用来做判断或预测的输入列。比如用材料密度、热处理温度、元素含量预测强度时，密度、温度和元素含量就是特征。选择特征时应优先选择与目标有合理关系、在预测时也能获得的列。

  • 适合作为特征：实验条件、成分、工艺参数、测量属性、类别描述。
  • 不建议作为特征：样本编号、文件名、备注、已经包含答案的信息、预测时无法获得的列。
  • 如果某列只是 ID，它可能让模型记住样本而不是学习规律，通常应设为忽略。

目标 Target

目标是希望模型预测的答案。回归任务的目标通常是数值，例如屈服强度、容量、价格；分类任务的目标通常是类别，例如材料类型、合格/不合格、工艺窗口标签。TabularLab 支持多目标任务，可以一次选择多个目标列。

忽略 Ignore

忽略表示该列不参与训练，也不会作为预测答案。对无关列、泄漏答案的列、过于杂乱的文本列或暂时不确定的列，可以先设为忽略，之后再逐步加入比较结果。

聚类任务中的字段角色

聚类没有目标列，因为它不是学习一个已知答案，而是根据特征相似性自动分组。聚类时应选择能表达样本差异的特征，例如物理性质、成分、工艺条件等。

5. 主要功能

数据导入与预览

导入后显示行数、列数、缺失值、数值列、类别列和字段摘要，并可以预览前 N 行数据。

字段角色管理

每列可以设置为特征、目标或忽略。目标列支持多选，适合多目标回归或多目标分类。

缺失值处理

支持数值均值/类别众数、数值中位数/类别众数、0/Unknown 三种策略。

类别编码

支持独热编码和序号编码，用于把文本或符号列转换为模型可使用的数值特征。

模型超参数

根据所选模型显示关键参数，例如 K、树深、树数量、学习率、正则化强度、训练轮数、DBSCAN 的 eps 和 minPts。

评估与导出

支持测试集评估、交叉验证 OOF 评估、图表保存、模型结果 JSON、预测 CSV 和处理后数据 CSV 导出。

6. 任务与模型说明

回归任务

回归用于预测连续数值。只要答案是一个可比较大小的数字，通常就是回归问题。例子包括根据材料成分预测屈服强度，根据电池材料参数预测容量，根据工艺参数预测性能。

  • Linear / Ridge Regression：线性模型，适合作为基础模型。Ridge 会通过正则化降低过拟合。
  • Lasso / ElasticNet：带正则化的线性模型，适合特征较多时尝试；Lasso 可能让部分特征权重接近 0。
  • Linear SVR：线性支持向量回归，对异常值相对稳健，但仍属于简化模型。
  • Decision Tree：决策树可表达非线性关系，容易理解，但过深时容易过拟合。
  • Random Forest：多棵树投票/平均，通常比单棵树更稳定，适合很多表格数据。
  • Gradient Boosting：逐步修正误差，常能获得较好精度，但参数更敏感。
  • KNN Regression：根据相似样本平均预测，直观但对特征尺度和无关特征较敏感。

分类任务

分类用于预测离散类别。只要答案是一个标签，通常就是分类问题。例子包括判断材料属于哪一类、工艺是否合格、样品是否落在可接受窗口内。

  • Softmax Logistic：多分类线性模型，适合做基线模型，结果较容易解释。
  • Linear SVM：线性分类模型，适合类别边界较清楚的数据。
  • Decision Tree：用一系列条件判断分类，直观但可能过拟合。
  • Random Forest：通常比单棵树更稳健，是表格分类的常用选择。
  • Gaussian Naive Bayes：速度快，适合做快速基线，但假设较强。
  • KNN Classifier：根据邻近样本投票分类，适合局部相似性明显的数据。

聚类任务

聚类用于无监督分组，也就是在没有目标答案的情况下，让模型根据样本相似性自动发现分群。聚类不需要目标列，也不需要训练/测试集划分；界面会使用全部选中特征数据进行聚类。

  • KMeans：需要指定簇数 K，适合希望得到固定数量分组的场景。
  • DBSCAN：根据密度发现簇，可以识别噪声点；适合簇形状不规则或存在离群点的数据。
  • Agglomerative Clustering：层次聚类，从每个样本单独成簇开始逐步合并，适合观察样本层次结构。

7. 参数设置与评估指标

预处理参数

  • 缺失值策略：均值/众数适合常规数据；中位数/众数对极端值更稳健；0/Unknown 适合 0 有明确含义或需要保留未知类别的情况。
  • 文本/符号编码：独热编码适合类别数量不太多且类别之间没有大小顺序的列；序号编码更紧凑，但会引入人为顺序。
  • 测试集比例：监督学习中用于留出一部分数据做最终评估。数据很少时比例不要过大，常见值为 0.2 到 0.3。
  • 随机种子：控制数据划分和部分模型的随机过程。固定种子可以让结果更容易复现。
  • 交叉验证与折数：K 折交叉验证会多次训练/验证，更充分利用数据；折数越大通常越慢。

模型超参数

  • K：在 KMeans/层次聚类中表示簇数，在 KNN 中表示参考的邻居数量。K 太小可能不稳定，太大可能过度平滑。
  • 树最大深度：限制决策树复杂度。深度越大越能拟合细节，也越容易过拟合。
  • 树数量：随机森林中的树越多通常越稳定，但运行更慢。
  • 学习率：每次更新的步长。较小学习率更稳但可能需要更多训练轮数。
  • 正则化强度：用于抑制过拟合。数值越大，模型越保守。
  • 训练轮数：线性/逻辑模型更新参数的次数。轮数太少可能欠拟合，太多会增加运行时间。
  • DBSCAN eps：判断样本互为邻居的距离阈值。过小会产生很多噪声，过大可能把不同簇合并。
  • DBSCAN minPts：形成核心点所需的最少邻居数。数值越大，对簇密度要求越高。

常见评估指标

  • MAE：平均绝对误差，越小越好，容易理解为平均偏差大小。
  • RMSE：均方根误差，越小越好，对大误差更敏感。
  • R2：回归拟合优度，越接近 1 通常越好；接近 0 表示相对基线改进有限。
  • Accuracy：分类准确率，表示预测正确比例；类别极不均衡时不要只看准确率。
  • Precision / Recall / F1：分别关注预测为某类时有多准、真实某类被找回多少、两者综合表现。
  • 混淆矩阵：展示每个真实类别被预测成各类别的数量，适合分析分类错误来源。
  • 聚类 inertia / sizes / noise：用于观察簇内紧密程度、每簇样本数和 DBSCAN 噪声点数量。

8. 标签页使用说明

  • 概览：查看数据集规模、缺失值情况、字段类型、唯一值、均值、标准差、最小值和最大值。

  • 结果：查看模型指标、当前评估模式、结果图、分类报告、聚类投影或线性系数。
  • 预测：输入单条样本进行预测，或上传新文件做批量预测并下载结果。
  • 数据表：预览原始数据，并导出经过缺失值和编码处理后的数据。
  • 关于：查看软件标题、作者、版本、发布日期、引用格式和 BibTeX。

  • 图表可保存为 PNG 文件。
  • 模型结果可保存为 JSON，包含任务类型、目标列、特征列、评估模式和指标。

10. 注意事项

  • TabularLab 面向中小型数据和快速实验，不建议直接用于百万行级生产训练。
  • 运行速度取决于本机性能。桌面 App 和浏览器方式都在本设备上处理数据。
  • 测试集指标只代表划分出的测试集表现；交叉验证指标代表 OOF 预测表现。
  • 批量预测文件必须包含训练时使用的全部特征列，其他列会保留在输出文件中。
  • 模型结果应结合领域知识判断。较高指标不一定表示模型可以直接用于真实生产决策。

TabularLab English User Manual

TabularLab is a local machine learning interface for tabular data. It can be used as a desktop app or opened in a browser, and helps users import data, define columns, preprocess features, train models, evaluate results, visualize outputs, make predictions, and export files.

Software Title TabularLab : A lightweight toolkit for tabular machine learning.
Author Dr. Bin Cao / 曹斌
Version v1.0.0, released on 2026-06-29
Download Apps Windows / macOS download page
Feedback If you have suggestions for improvement, please open an issue on GitHub Issues.

1. Purpose

The purpose of this interface is to make tabular machine learning accessible without installing Python, R, or a backend service. In the desktop app, dependencies and example datasets are bundled; in browser use, calculations run locally in the browser. The tool is suitable for teaching, materials-data exploration, prototyping, model comparison, and quick experiments.

  • Lower the entry barrier for tabular ML workflows: load a file, choose a task, and run analysis.
  • Support regression, classification, and clustering in one interface.
  • Combine preprocessing, train/test splitting, cross validation, metrics, charts, predictions, and exports.
  • Help users understand how each setting affects model outputs.

2. Quick Start

Open the interface

Download the Windows or macOS app from the release page and run it directly. You can also open index.html in a browser; for browser use, a local static server is recommended, such as python3 -m http.server 8000, then open http://localhost:8000/.

Import data

Drop a CSV, TSV, TXT, XLSX, or XLS file into the upload area, or choose a default dataset and click “Load Sample”.

Select a task

Choose regression, classification, or clustering. Regression predicts continuous values, classification predicts labels, and clustering discovers groups without target labels.

Assign column roles

In the column-role table, set each column as a feature, target, or ignored column. Clustering does not require a target column.

Configure processing and training

Choose a missing-value strategy, text encoding method, model, and model hyperparameters. For supervised tasks, configure test size, random seed, cross validation, and folds.

Run and inspect results

Click “Run Analysis”, then inspect metrics, charts, predictions, and exports in the Results, Visuals, Predict, and Data Table tabs.

3. Data Sources and Requirements

Supported formats

  • .csv, .tsv, .txt: delimited text tables.
  • .xlsx, .xls: Excel files parsed by the bundled SheetJS library.

Data recommendations

  • The first row should contain clear and unique column names.
  • Regression targets should be numeric, such as strength, price, capacity, or temperature.
  • Classification targets should be labels, such as material class, pass/fail, or process-window status.
  • Clustering tasks do not need a target column; selected features should describe meaningful sample differences.
  • Missing values can remain in the file. They are handled according to the selected strategy.

4. Column Roles: Feature, Target, and Ignore

A tabular machine learning dataset is made of rows and columns. Each row is a sample, and each column contains one type of information. Before running a model, TabularLab needs to know how each column should be used.

Feature

Features are the input columns used by the model to make a prediction or discover patterns. For example, if you predict strength from density, heat-treatment temperature, and composition, those input columns are features. Good features should be related to the target and available when future predictions are made.

  • Good feature candidates: experimental conditions, composition, process parameters, measured properties, and category descriptors.
  • Usually poor feature candidates: sample ID, file name, free-text notes, columns that already contain the answer, or columns unavailable at prediction time.
  • If a column is only an ID, it may encourage the model to memorize samples instead of learning a useful rule. It is usually better to ignore it.

Target

The target is the answer you want the model to predict. Regression targets are numeric values, such as yield strength, capacity, or price. Classification targets are labels, such as material class, pass/fail, or process-window status. TabularLab also supports multiple target columns.

Ignore

Ignored columns are not used for training and are not predicted. Use this for irrelevant columns, answer-leakage columns, messy text columns, or columns you are not ready to include.

Roles in Clustering

Clustering has no target column because it does not learn from known answers. It groups samples by feature similarity. Choose features that describe meaningful differences between samples, such as physical properties, composition, or processing conditions.

5. Main Functions

Data import and preview

After import, the interface shows rows, columns, missing values, numeric columns, categorical columns, column summaries, and the first N preview rows.

Column role management

Each column can be marked as feature, target, or ignored. Multiple targets are supported for multi-target regression or classification.

Missing-value handling

Supported strategies include numeric mean/category mode, numeric median/category mode, and 0/Unknown.

Categorical encoding

One-hot and ordinal encoding convert text or symbolic columns into numeric features for machine learning models.

Model hyperparameters

The interface displays key parameters for the selected model, such as K, tree depth, tree count, learning rate, regularization, epochs, DBSCAN eps, and minPts.

Evaluation and export

TabularLab supports test-set evaluation, cross-validation OOF evaluation, chart export, model-result JSON, prediction CSV, and processed-data CSV.

6. Tasks and Models

Regression

Regression predicts continuous numeric values. If the answer is a number with magnitude and units, it is usually a regression problem. Examples include predicting yield strength from composition, battery capacity from material parameters, or performance from process settings.

  • Linear / Ridge Regression: A simple baseline model. Ridge regularization helps reduce overfitting.
  • Lasso / ElasticNet: Regularized linear models. They are useful when there are many features; Lasso can shrink some weights toward zero.
  • Linear SVR: A linear support-vector regressor. It can be more robust to some errors but is still a simplified model.
  • Decision Tree: Captures nonlinear rules and is easy to understand, but deep trees can overfit.
  • Random Forest: Combines many trees and is usually more stable than a single tree for tabular data.
  • Gradient Boosting: Builds models sequentially to correct previous errors. It can be accurate but is more sensitive to parameters.
  • KNN Regression: Predicts from nearby samples. It is intuitive but sensitive to feature scaling and irrelevant features.

Classification

Classification predicts discrete labels. If the answer is a class name or category, it is usually a classification problem. Examples include material family, pass/fail status, or whether a sample is inside a process window.

  • Softmax Logistic: A linear multi-class classifier. It is a useful baseline and relatively easy to interpret.
  • Linear SVM: A linear classifier that works well when class boundaries are fairly clear.
  • Decision Tree: Uses a sequence of rules for classification. It is readable but can overfit.
  • Random Forest: Often more robust than a single tree and a common choice for tabular classification.
  • Gaussian Naive Bayes: Fast and useful as a quick baseline, but it makes strong assumptions.
  • KNN Classifier: Classifies by voting among nearby samples. It works best when local similarity is meaningful.

Clustering

Clustering discovers groups without labels. It is useful when you do not already know the answer but want to explore natural groupings in the data. It does not need a target column or train/test split; all selected feature rows are used for clustering.

  • KMeans: Requires a cluster count K. It is useful when you want a fixed number of groups.
  • DBSCAN: Finds dense regions and can label noise points. It is useful for irregular cluster shapes or outliers.
  • Agglomerative Clustering: A hierarchical method that starts from individual samples and merges them step by step.

7. Parameters and Metrics

Preprocessing Parameters

  • Missing-value strategy: Mean/mode is a common default; median/mode is more robust to outliers; 0/Unknown is useful when zero or unknown has a meaningful interpretation.
  • Text/symbol encoding: One-hot encoding is suitable for categories without natural order and moderate category counts. Ordinal encoding is compact but introduces an artificial order.
  • Test size: Holds out part of supervised data for final evaluation. Common values are around 0.2 to 0.3.
  • Random seed: Controls splits and some model randomness. A fixed seed makes results easier to reproduce.
  • Cross validation and folds: K-fold cross validation trains and validates multiple times, using data more fully. More folds usually take longer.

Model Hyperparameters

  • K: The cluster count for KMeans/agglomerative clustering, or the neighbor count for KNN. Too small can be unstable; too large can oversmooth.
  • Max tree depth: Limits tree complexity. Deeper trees can fit details but may overfit.
  • Tree count: More random-forest trees are usually more stable but slower.
  • Learning rate: Controls the update size. Smaller values are steadier but may need more training rounds.
  • Regularization: Penalizes overly complex models. Larger values make the model more conservative.
  • Epochs: Number of parameter-update passes for linear/logistic models. Too few can underfit; too many increase runtime.
  • DBSCAN eps: Distance threshold for neighborhood membership. Too small creates many noise points; too large may merge separate clusters.
  • DBSCAN minPts: Minimum neighbors required for a core point. Larger values require denser clusters.

Common Metrics

  • MAE: Mean absolute error. Lower is better and easy to read as average error size.
  • RMSE: Root mean squared error. Lower is better and more sensitive to large errors.
  • R2: Regression goodness-of-fit. Values closer to 1 are usually better; values near 0 indicate limited improvement over a baseline.
  • Accuracy: Fraction of correct classifications. For imbalanced classes, do not rely on accuracy alone.
  • Precision / Recall / F1: Precision asks how reliable positive predictions are; recall asks how many real positives were found; F1 balances both.
  • Confusion matrix: Shows how true classes are predicted, helping identify error patterns.
  • Clustering inertia / sizes / noise: Help inspect compactness, cluster sizes, and DBSCAN noise points.

8. Tab Guide

  • Overview: Inspect dataset size, missing values, column types, unique values, mean, standard deviation, minimum, and maximum.
  • Visuals: Draw histograms, scatter plots, correlation heatmaps, and target plots, then save charts as PNG files.
  • Results: Inspect model metrics, evaluation mode, result charts, classification report, clustering projection, or linear coefficients.
  • Predict: Enter one sample for prediction, or upload a new file for batch prediction and download the output.
  • Data Table: Preview raw data and export processed data after missing-value handling and encoding.
  • About: View software title, author, version, release date, citation, and BibTeX.

9. Saving and Citation

  • Charts can be saved as PNG files.
  • Model results can be saved as JSON with task type, targets, features, evaluation mode, and metrics.
  • Prediction outputs can be saved as CSV for downstream analysis or reporting.
  • Processed data can be saved as CSV for reproducibility or use in other tools.

10. Notes

  • TabularLab is designed for small to medium datasets and quick experiments, not million-row production training.
  • Runtime depends on the local computer. Browser use is affected by browser performance, while desktop app use is affected by the bundled Electron runtime and system resources.
  • Test metrics represent the held-out test split. Cross-validation metrics represent out-of-fold predictions.
  • Batch prediction files must contain every feature column used during training. Other columns are preserved in the output.
  • Model results should be interpreted with domain knowledge. Good metrics do not automatically mean a model is ready for production decisions.
